 Serverless Image Processing Pipeline (AWS | Terraform | Lambda)
Built a backend-only serverless image processing system that automatically transforms images uploaded to Amazon S3 using AWS Lambda.

🎯 Architecture Overview
When a user uploads an image:
📤 Image uploaded to S3 Upload Bucket
⚡ S3 Event Trigger invokes Lambda automatically
🖼️ Lambda processes image into multiple optimized formats
📦 Processed images stored in S3 Processed Bucket

Fully automated. No servers. No manual intervention.

🧱 Core AWS Services Used
🗂️ Amazon S3 – Storage for original & processed images
⚡ AWS Lambda – Event-driven image processor
📦 Lambda Layer (Pillow 10.4.0) – Image manipulation library
🔔 S3 Event Notifications – Auto-trigger on object upload
📊 Amazon CloudWatch – Logging & monitoring
🏗️ Terraform – Infrastructure as Code (IaC)

🎨 Image Variants Generated
For every uploaded image:
✅ Compressed JPEG (85% quality)
✅ Low Quality JPEG (60%)
✅ WebP (modern optimized format)
✅ PNG (lossless format)
✅ Thumbnail (200x200)

Example:
Original: 500 KB
Compressed: ~120 KB
Low Quality: ~80 KB
WebP: ~95 KB
Thumbnail: ~15 KB

⚙️ Key Features
🔁 Fully event-driven architecture
🔐 Private S3 buckets (no public access)
🔒 AES256 server-side encryption
🛡️ IAM least-privilege permissions
📦 Infrastructure automated via Terraform
♻️ Force destroy enabled for dev cleanup
🌍 Region independent deployment

📊 Performance Metrics
⚡ Cold start: ~470ms
🚀 Warm execution: 300–600ms
🧠 Memory usage: ~113MB (1024MB allocated)
⏱️ ~100ms per image variant

Efficient and production-ready.

Highly scalable and cost-effective.
